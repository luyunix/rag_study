## 这节到底讲什么

**核心问题：为什么说 Agent 是大模型的“手脚”？**

这节直接回答“为什么说 Agent 是大模型的“手脚”？”。老师的结论可以整理成五点：第一，大脑：LLM 理解目标并规划下一步；第二，工具：搜索、数据库、计算与业务 API；第三，行动：用结构化参数真正调用外部能力；第四，观察：读取工具结果后更新判断；第五，约束：权限、超时、成本、错误与停止条件。下面逐项解释每一点的含义和作用。

## 用一个例子串起来

用户问制度问题时调用制度知识库，问公司投资关系时调用金融图谱。Agent Router 先判断意图、选择工具、读取结果，再决定是否继续调用或给出答案。
